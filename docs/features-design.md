# KEY GENE 新功能需求方案

> 基于现有技术栈：纯静态 HTML/JS/CSS + Cloudflare Worker API 代理 + Leaflet.js + Chart.js
> 新功能优先整合进现有页面，只在必要时新建独立页面
> 三语支持（EN/ZH/KO）

---

## 设计原则

1. **能整合就不新建页面** — 减少导航复杂度，缩短用户路径
2. **搜过的玩家数据复用** — 不让用户重复输入，标签页之间共享查询上下文
3. **纯前端优先** — 不依赖 API 的工具先做，稳定性高、加载快

---

## 现有功能改进

### 改进一：Ban Check 去掉独立输入框

**现状问题**：用户在 Stats 搜索框搜了玩家后，切到 Ban Check 标签页还要再手动输入一次名字。

**改进方案**：
- 搜索玩家后，切到 Ban Check 标签页**自动查询当前玩家**
- 去掉 Ban Check 标签页内的独立输入框和查询按钮
- 查询结果直接展示，和其他标签页（Recent Matches、Weapons 等）体验一致
- 如果用户未搜索就直接点 Ban Check 标签，显示提示"请先搜索玩家"

### 改进二：Compare 标签页加分享海报

**现状**：Compare 已有雷达图 + 逐项对比，但没有分享功能。

**改进方案**：
- 对比结果底部加「生成 VS 海报」按钮
- html2canvas 截取对比区域，生成带 KEY GENE logo 的分享图片
- 这就覆盖了原"对决 PK"独立页面的需求，不再新建 versus.html

## 功能一：灵敏度配置分享器 (`sensitivity.html`)

### 定位
PUBG 玩家换鼠标/显示器后第一件事就是搜灵敏度换算。搜索引擎长尾流量入口。

### 功能模块

**1. 灵敏度计算器**
- 输入项：DPI、游戏内总灵敏度、垂直灵敏度倍率
- 倍镜灵敏度：1x / 2x / 3x / 4x / 6x / 8x / 15x（各自独立滑块）
- 实时计算并显示：
  - eDPI（有效灵敏度 = DPI × 游戏灵敏度）
  - 360° 转身距离（cm/360）
  - 每个倍镜的实际 cm/360
- 换鼠标 DPI 转换：输入新 DPI，自动算出等效灵敏度

**2. 职业选手配置库**
- 预置 15-20 个知名选手的灵敏度配置（来源：prosettings.net）
- 选手卡片：头像、战队、DPI、灵敏度、倍镜设置
- 点击「应用此配置」→ 计算器自动填入该选手的数值
- 按 DPI 范围 / 风格（高灵敏/低灵敏）筛选

**3. 配置分享**
- 「生成分享卡片」→ html2canvas 生成带 KEY GENE logo 的配置图片
- 配置编码为 URL 参数：`sensitivity.html?d=800&s=50&v=100&1x=50&2x=...`
- 分享链接打开后自动还原配置

### 页面布局
```
┌──────────────────────────────────────┐
│ Header (shared)                       │
├──────────────────────────────────────┤
│  [计算器区域]          [职业选手库]    │
│  DPI: [___]            ┌──────────┐  │
│  灵敏度: [===slider===] │ Pio      │  │
│  垂直倍率: [===]       │ DPI:400  │  │
│  ─────────────         │ Sens:50  │  │
│  1x: [===] cm/360: 28  │ [应用]   │  │
│  2x: [===] cm/360: 42  ├──────────┤  │
│  3x: [===] cm/360: 56  │ NiceWigg │  │
│  ...                   │ ...      │  │
│  ─────────────         └──────────┘  │
│  eDPI: 40000                         │
│  360°: 27.5 cm                       │
│  [生成分享卡] [复制链接]              │
├──────────────────────────────────────┤
│ Footer (shared)                       │
└──────────────────────────────────────┘
```

### 技术要点
- 纯前端计算，无需 API
- 公式：cm/360 = (2.54 × 360) / (DPI × sens / 50 × multiplier)
- 选手数据存储在页面内 JS 对象中（静态，手动更新）
- html2canvas 已在 stats.html 引入，复用

## 功能二：航线跳伞模拟器 (`flight.html`)

### 定位
每局开局决策工具。在地图上可视化航线，推荐最佳跳伞点。

### 功能模块

**1. 航线绘制**
- 在 Leaflet 地图上点击两点画航线（起点 → 终点）
- 显示航线距离、飞行时间（飞机速度 ~130km/h）
- 航线沿途标注 10 秒间隔的飞机位置

**2. 跳伞范围可视化**
- 以航线为中心，显示最大到达范围的半透明椭圆
  - 垂直跳伞（直落）：~750m 横向距离
  - 平飞跳伞（45°俯角）：~1.5km
  - 极限滑翔（开伞后滑翔）：~2.5km
- 三层同心环：近距离（绿）、中距离（黄）、远距离（红）

**3. 热门资源点推荐**
- 预置每张地图的热门降落点（20-30 个/地图）
- 根据航线自动计算：
  - 每个资源点到航线的最近距离
  - 是否在跳伞范围内（近/中/远）
  - 排序推荐：兼顾距离、物资丰富度、竞争程度
- 资源点标注：名称、物资评级（S/A/B/C）、预计竞争人数

**4. 分享**
- 截图分享当前航线 + 推荐方案

### 页面布局
- 复用 maps.html 的 Leaflet 地图 + pubgCRS + 瓦片
- 左侧：地图选择 sidebar（复用 maps.html 样式）
- 右侧面板：航线信息 + 推荐跳伞点列表
- 底部工具栏：画航线、清除、分享

### 技术要点
- 复用 maps.html 的地图基础设施
- 热门降落点数据：新建 `assert/maps/dropspots.json`
- 距离计算：点到线段最短距离公式
- 无需 API，纯前端

## 功能三：队伍数据对比 (`team.html`)

### 定位
4 排玩家输入 4 个队友 ID，生成团队分析报告，帮助分配角色。

### 功能模块

**1. 队伍输入**
- 4 个输入框（Player 1-4），支持 URL 参数预填
- 选择赛季（当前/历史）和模式（TPP/FPP）
- 一键查询

**2. 团队雷达图**
- 6 维雷达图（Chart.js radar）：
  - 击杀能力（场均击杀）
  - 生存能力（Top 10 率）
  - 输出能力（场均伤害）
  - 辅助能力（场均助攻）
  - 胜率（吃鸡率）
  - 经验值（总场次）
- 4 个玩家叠加在同一雷达图上，不同颜色

**3. 角色推荐**
- 根据数据自动推荐角色分工：
  - 突破手（击杀最高）
  - 指挥官（胜率 + 生存最均衡）
  - 狙击手（最远击杀距离最大）
  - 辅助位（助攻最高）
- 每个玩家显示匹配度百分比

**4. 团队总结卡片**
- html2canvas 生成分享图：团队雷达图 + 4 人 ID + 角色分配
- URL 参数分享：`team.html?p1=xxx&p2=xxx&p3=xxx&p4=xxx`

### 技术要点
- 复用 stats.html 的 PUBG API 调用逻辑（queueFetch、lifetime stats）
- Chart.js radar chart（stats.html 已引入 Chart.js）
- 4 个玩家并行请求，Promise.all
- 角色推荐算法：各维度 z-score 标准化后加权匹配

## 功能四：赛季总结报告（stats.html `Wrapped` 标签页）

### 定位
类似 Spotify Wrapped，搜索玩家后切到 Wrapped 标签页生成赛季总结海报。自带社交传播。

### 用户路径
1. 在 stats.html 搜索玩家（复用现有搜索框）
2. 点击 `Wrapped` 标签页
3. 自动生成赛季总结报告（无需二次输入）
4. 截图/分享

### 功能模块

**1. 数据采集（复用已有 API 调用）**
- lifetime stats：总击杀、总场次、K/D、胜率、Top10 率
- 最近 20 场 telemetry：最常用武器、武器击杀占比
- 最远击杀距离、最高单场击杀
- 常去的地图分布

**2. 卡片式报告（纵向滚动，5 个卡片）**
- 卡片 1：「赛季回顾」标题 + 玩家名 + 赛季 + 段位
- 卡片 2：核心数据大字（总击杀、场次、K/D）+ 分位数排名（"超过了 X% 的玩家"）
- 卡片 3：最爱武器 TOP 3 + 击杀占比饼图
- 卡片 4：地图偏好雷达图 + 最远击杀
- 卡片 5：趣味标签（根据数据给标签："钢枪王"、"活到最后的人"、"狙击之神"等）+ KEY GENE logo + 分享按钮

**3. 分享**
- 「生成海报」按钮 → html2canvas 截取整个报告区域
- 竖版海报比例（适合微信/Discord 分享）
- 带 KEY GENE 品牌水印

### 技术要点
- 不新建页面，作为 stats.html 的第 7 个标签页
- 复用已有的 `queueFetch`、`renderMatches`、`renderWeapons` 数据
- 分位数对照表：硬编码经验值（击杀 >2000 = top 5% 之类）
- 趣味标签算法：根据 K/D、胜率、最远击杀、爆头率等维度的相对强弱自动分配
- 数字动画：requestAnimationFrame 递增

## 功能五：武器数据库 (`weapons.html`)

### 定位
每次版本更新玩家都搜"哪把枪被加强了"。SEO 长尾流量。

### 功能模块

**1. 武器列表**
- 按分类筛选：步枪 / 狙击 / 冲锋枪 / 霰弹枪 / 手枪 / 近战
- 搜索框
- 卡片式展示：武器图标 + 名称 + 核心数据

**2. 武器详情**
- 基础数据：伤害、射速、弹速、装弹时间、弹夹容量
- TTK 图表（不同距离/不同护甲下的击杀时间）
- 后坐力模式图（前 10 发的弹道散布图）
- 可装配件列表

**3. 武器对比**
- 选择 2-3 把武器并排对比
- 雷达图对比（伤害、射速、稳定性、射程、机动性）
- TTK 对比曲线（Chart.js line chart）

**4. Patch Notes 标注**
- 每把武器标注最近的改动（如 "31.1: 伤害 43→41"）
- 排序选项：按最近更新时间

### 数据来源
- 武器数据存储为静态 JSON：`assert/data/weapons.json`
- 手动维护（PUBG 更新不频繁，几个月一次大改）
- 数据来源：PUBG 官方 patch notes + pubg.fandom.com

### 技术要点
- 纯前端，无需 API
- Chart.js 雷达图 + 折线图
- 武器图标：使用游戏内截图或简单 SVG 剪影

## 功能七：投掷物计算器 (`throwables.html`)

### 定位
实用型工具，帮助玩家掌握手雷/烟雾弹的最佳投掷位置。

### 功能模块

**1. 投掷物信息卡**
- 手榴弹：引信 5 秒，最大投掷距离 ~60m，伤害范围 15m
- 烟雾弹：引信 3 秒，持续 15 秒，烟雾半径 ~10m
- 闪光弹：引信 2.5 秒，致盲范围 100°扇形
- 燃烧瓶：着地即爆，持续 7 秒，范围 ~5m
- 粘性炸弹：粘附后 3 秒爆炸

**2. 地图投掷辅助**
- 在 Leaflet 地图上：
  - 点击设置投掷起点
  - 点击设置目标终点
  - 显示距离 + 是否在投掷范围内
  - 如果超出范围，显示需要多少前移
- 常用投掷位整合：
  - 预置每张地图的经典投掷位（如 Erangel 学校天台→楼下窗户）
  - 每个位置附带教学说明

**3. 投掷技巧**
- 三种投掷方式的区别（上手投、下手投、中手投）
- 各方式的距离/弧度对比图

### 技术要点
- 复用 maps.html 的 Leaflet 地图
- 投掷数据存储为静态 JSON
- 距离计算复用现有 km 换算公式
- 弧线可视化：用 Leaflet polyline 画抛物线轨迹

## 导航规划

### 页面结构总览

**整合进现有页面的功能（不新建页面）：**
- 赛季总结 → stats.html 新标签页 `Wrapped`

**新建独立页面（4 个）：**
- sensitivity.html → 灵敏度配置
- weapons.html → 武器数据库
- team.html → 队伍分析
- flight.html → 航线模拟

```
Stats ▼          Maps ▼           Tools ▼           Quiz

├─ 队伍分析       └─ 航线模拟       └─ 武器数据库
```
