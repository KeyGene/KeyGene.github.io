# KEY GENE 新功能需求方案

> 基于现有技术栈：纯静态 HTML/JS/CSS + Cloudflare Worker API 代理 + Leaflet.js + Chart.js
> 所有功能均为独立页面，复用 shared.css / shared.js，三语支持（EN/ZH/KO）

## 功能一：灵敏度配置分享器 (`sensitivity.html`)

### 定位
PUBG 玩家换鼠标/显示器后第一件事就是搜灵敏度换算。搜索引擎长尾流量入口。

### 功能模块

**1. 灵敏度计算器**
- 输入项：DPI、游戏内总灵敏度、垂直灵敏度倍率
- 倍镜灵敏度：1x / 2x / 3x / 4x / 6x / 8x / 15x（各自独立滑块）
- 实时计算并显示：
  - eDPI（有效灵敏度 = DPI × 游戏灵敏度）
  - 360° 转身距离（cm/360）
  - 每个倍镜的实际 cm/360
- 换鼠标 DPI 转换：输入新 DPI，自动算出等效灵敏度

**2. 职业选手配置库**
- 预置 15-20 个知名选手的灵敏度配置（来源：prosettings.net）
- 选手卡片：头像、战队、DPI、灵敏度、倍镜设置
- 点击「应用此配置」→ 计算器自动填入该选手的数值
- 按 DPI 范围 / 风格（高灵敏/低灵敏）筛选

**3. 配置分享**
- 「生成分享卡片」→ html2canvas 生成带 KEY GENE logo 的配置图片
- 配置编码为 URL 参数：`sensitivity.html?d=800&s=50&v=100&1x=50&2x=...`
- 分享链接打开后自动还原配置

### 页面布局
```
┌──────────────────────────────────────┐
│ Header (shared)                       │
├──────────────────────────────────────┤
│  [计算器区域]          [职业选手库]    │
│  DPI: [___]            ┌──────────┐  │
│  灵敏度: [===slider===] │ Pio      │  │
│  垂直倍率: [===]       │ DPI:400  │  │
│  ─────────────         │ Sens:50  │  │
│  1x: [===] cm/360: 28  │ [应用]   │  │
│  2x: [===] cm/360: 42  ├──────────┤  │
│  3x: [===] cm/360: 56  │ NiceWigg │  │
│  ...                   │ ...      │  │
│  ─────────────         └──────────┘  │
│  eDPI: 40000                         │
│  360°: 27.5 cm                       │
│  [生成分享卡] [复制链接]              │
├──────────────────────────────────────┤
│ Footer (shared)                       │
└──────────────────────────────────────┘
```

### 技术要点
- 纯前端计算，无需 API
- 公式：cm/360 = (2.54 × 360) / (DPI × sens / 50 × multiplier)
- 选手数据存储在页面内 JS 对象中（静态，手动更新）
- html2canvas 已在 stats.html 引入，复用

## 功能二：航线跳伞模拟器 (`flight.html`)

### 定位
每局开局决策工具。在地图上可视化航线，推荐最佳跳伞点。

### 功能模块

**1. 航线绘制**
- 在 Leaflet 地图上点击两点画航线（起点 → 终点）
- 显示航线距离、飞行时间（飞机速度 ~130km/h）
- 航线沿途标注 10 秒间隔的飞机位置

**2. 跳伞范围可视化**
- 以航线为中心，显示最大到达范围的半透明椭圆
  - 垂直跳伞（直落）：~750m 横向距离
  - 平飞跳伞（45°俯角）：~1.5km
  - 极限滑翔（开伞后滑翔）：~2.5km
- 三层同心环：近距离（绿）、中距离（黄）、远距离（红）

**3. 热门资源点推荐**
- 预置每张地图的热门降落点（20-30 个/地图）
- 根据航线自动计算：
  - 每个资源点到航线的最近距离
  - 是否在跳伞范围内（近/中/远）
  - 排序推荐：兼顾距离、物资丰富度、竞争程度
- 资源点标注：名称、物资评级（S/A/B/C）、预计竞争人数

**4. 分享**
- 截图分享当前航线 + 推荐方案

### 页面布局
- 复用 maps.html 的 Leaflet 地图 + pubgCRS + 瓦片
- 左侧：地图选择 sidebar（复用 maps.html 样式）
- 右侧面板：航线信息 + 推荐跳伞点列表
- 底部工具栏：画航线、清除、分享

### 技术要点
- 复用 maps.html 的地图基础设施
- 热门降落点数据：新建 `assert/maps/dropspots.json`
- 距离计算：点到线段最短距离公式
- 无需 API，纯前端

## 功能三：队伍数据对比 (`team.html`)

### 定位
4 排玩家输入 4 个队友 ID，生成团队分析报告，帮助分配角色。

### 功能模块

**1. 队伍输入**
- 4 个输入框（Player 1-4），支持 URL 参数预填
- 选择赛季（当前/历史）和模式（TPP/FPP）
- 一键查询

**2. 团队雷达图**
- 6 维雷达图（Chart.js radar）：
  - 击杀能力（场均击杀）
  - 生存能力（Top 10 率）
  - 输出能力（场均伤害）
  - 辅助能力（场均助攻）
  - 胜率（吃鸡率）
  - 经验值（总场次）
- 4 个玩家叠加在同一雷达图上，不同颜色

**3. 角色推荐**
- 根据数据自动推荐角色分工：
  - 突破手（击杀最高）
  - 指挥官（胜率 + 生存最均衡）
  - 狙击手（最远击杀距离最大）
  - 辅助位（助攻最高）
- 每个玩家显示匹配度百分比

**4. 团队总结卡片**
- html2canvas 生成分享图：团队雷达图 + 4 人 ID + 角色分配
- URL 参数分享：`team.html?p1=xxx&p2=xxx&p3=xxx&p4=xxx`

### 技术要点
- 复用 stats.html 的 PUBG API 调用逻辑（queueFetch、lifetime stats）
- Chart.js radar chart（stats.html 已引入 Chart.js）
- 4 个玩家并行请求，Promise.all
- 角色推荐算法：各维度 z-score 标准化后加权匹配

## 功能四：赛季总结报告 (`wrapped.html`)

### 定位
类似 Spotify Wrapped，玩家输入 ID 生成赛季数据总结海报，自带社交传播。

### 功能模块

**1. 数据采集**
- 输入玩家 ID → 拉取当前赛季数据
- 采集维度：
  - 总击杀数、总场次、总伤害
  - 最常用武器（从最近 20 场比赛的 telemetry 统计）
  - 最远击杀距离
  - 最高单场击杀
  - 常去的地图
  - 胜率变化趋势

**2. 翻页式报告（5-6 页）**
- 第 1 页：「你的赛季回顾」标题 + 玩家名 + 赛季标签
- 第 2 页：核心数据（击杀数大字 + "超过了 X% 的玩家"）
- 第 3 页：最爱武器 + 武器击杀占比饼图
- 第 4 页：地图偏好 + 最远击杀
- 第 5 页：角色标签（根据数据给一个趣味标签：如"钢枪王"、"苟到最后的男人"）
- 第 6 页：总结 + KEY GENE logo + 分享按钮

**3. 分享**
- 每一页都可单独截图
- 最终页截图包含所有核心数据 + KEY GENE 品牌
- 生成分享链接

### 页面布局
- 全屏翻页式（垂直滑动或左右滑动）
- 深色背景 + 大字数据 + 动画数字滚动
- 移动端优先设计（竖屏海报比例）

### 技术要点
- PUBG API：lifetime stats + 最近比赛 telemetry
- 数字动画：CSS counter 或 requestAnimationFrame 递增
- "超过 X% 的玩家"：需要预置一个大致的分位数对照表
  - 可以从排行榜 API 抽样估算，或硬编码经验值
- html2canvas 逐页截图

## 功能五：对决 PK 页 (`versus.html`)

### 定位
玩家互相攀比的刚需。输入两个 ID，生成对比图。

### 功能模块

**1. 双人输入**
- 左右两个输入框 + 中间 VS 标志
- 支持 URL 参数：`versus.html?p1=xxx&p2=xxx`

**2. 逐项对比**
- 对比维度（每项一行，进度条对比）：
  - K/D 比率
  - 场均伤害
  - 吃鸡率
  - Top 10 率
  - 场均击杀
  - 最远击杀
  - 爆头率
  - 总场次
- 每项高亮胜出者（红色高亮）
- 底部总分：胜出项目数 X vs Y

**3. 结论**
- 自动生成一句话评价："PlayerA 更擅长正面刚枪，PlayerB 更稳定"
- 趣味标签对比（复用 wrapped 的标签逻辑）

**4. 分享**
- html2canvas 生成 VS 对比海报
- URL 分享

### 技术要点
- stats.html 的 Compare 标签页已有类似逻辑，可复用
- 两个玩家并行查询
- 进度条宽度 = 该项数值占两人总和的比例

### 与现有功能的关系
- stats.html 已有 Compare 标签页，但功能较基础
- versus.html 是独立的、更视觉化的 PK 体验
- 可以从 stats.html Compare 标签页加链接跳转

## 功能六：武器数据库 (`weapons.html`)

### 定位
每次版本更新玩家都搜"哪把枪被加强了"。SEO 长尾流量。

### 功能模块

**1. 武器列表**
- 按分类筛选：步枪 / 狙击 / 冲锋枪 / 霰弹枪 / 手枪 / 近战
- 搜索框
- 卡片式展示：武器图标 + 名称 + 核心数据

**2. 武器详情**
- 基础数据：伤害、射速、弹速、装弹时间、弹夹容量
- TTK 图表（不同距离/不同护甲下的击杀时间）
- 后坐力模式图（前 10 发的弹道散布图）
- 可装配件列表

**3. 武器对比**
- 选择 2-3 把武器并排对比
- 雷达图对比（伤害、射速、稳定性、射程、机动性）
- TTK 对比曲线（Chart.js line chart）

**4. Patch Notes 标注**
- 每把武器标注最近的改动（如 "31.1: 伤害 43→41"）
- 排序选项：按最近更新时间

### 数据来源
- 武器数据存储为静态 JSON：`assert/data/weapons.json`
- 手动维护（PUBG 更新不频繁，几个月一次大改）
- 数据来源：PUBG 官方 patch notes + pubg.fandom.com

### 技术要点
- 纯前端，无需 API
- Chart.js 雷达图 + 折线图
- 武器图标：使用游戏内截图或简单 SVG 剪影

## 功能七：投掷物计算器 (`throwables.html`)

### 定位
实用型工具，帮助玩家掌握手雷/烟雾弹的最佳投掷位置。

### 功能模块

**1. 投掷物信息卡**
- 手榴弹：引信 5 秒，最大投掷距离 ~60m，伤害范围 15m
- 烟雾弹：引信 3 秒，持续 15 秒，烟雾半径 ~10m
- 闪光弹：引信 2.5 秒，致盲范围 100°扇形
- 燃烧瓶：着地即爆，持续 7 秒，范围 ~5m
- 粘性炸弹：粘附后 3 秒爆炸

**2. 地图投掷辅助**
- 在 Leaflet 地图上：
  - 点击设置投掷起点
  - 点击设置目标终点
  - 显示距离 + 是否在投掷范围内
  - 如果超出范围，显示需要多少前移
- 常用投掷位整合：
  - 预置每张地图的经典投掷位（如 Erangel 学校天台→楼下窗户）
  - 每个位置附带教学说明

**3. 投掷技巧**
- 三种投掷方式的区别（上手投、下手投、中手投）
- 各方式的距离/弧度对比图

### 技术要点
- 复用 maps.html 的 Leaflet 地图
- 投掷数据存储为静态 JSON
- 距离计算复用现有 km 换算公式
- 弧线可视化：用 Leaflet polyline 画抛物线轨迹

## 导航规划

### 新增页面导航结构
现有导航已经有 5 个链接（Stats, Maps, Quiz + Join Us），不能无限加。

**方案：二级导航分组**

```
Stats ▼          Maps ▼           Tools ▼           Quiz
├─ 战绩查询       ├─ 互动地图       ├─ 灵敏度配置

├─ 队伍分析       └─ 投掷物计算     └─ 赛季总结

**Header 实现**：
- PC 端：hover 显示下拉菜单
- 移动端：点击展开子菜单

```
